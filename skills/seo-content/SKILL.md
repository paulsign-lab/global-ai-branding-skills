# seo-content — SEO·GEO 콘텐츠 자동 생성

구글 Opal로 만들던 콘텐츠 자동화 워크플로우(주제 발굴→아웃라인→본문→번역→메타)를
**클로드코드 스킬로 박제**하고, 단순 SERP 스크래핑 대신 **Semrush MCP의 체계적
검색 데이터**로 강화했다. 키워드·타깃 국가만 넣으면 검색엔진과 AI(GEO) 양쪽에
최적화된 콘텐츠 자산이 나온다.

> **강의 어필 포인트:** "워크플로우를 직접 만들어 각 단계를 편집하셔도 좋고(Opal),
> 이 스킬로 편하게 SEO 최적화 콘텐츠를 바로 받아보셔도 됩니다."

## 언제 사용하나
- 타깃 국가의 검색 수요에 맞는 SEO 블로그 콘텐츠가 필요할 때
- 키워드 리서치→글쓰기→메타태그를 한 번에 처리하고 싶을 때

## 사전 준비
- **Semrush MCP 연결**(키워드·검색량·질문형 키워드 조사용). 미연결 시 SERP 조사로 대체 가능.

## 입력값
- **메인 키워드** (예: korean sunscreen SPF 50)
- **타깃 국가** (예: 미국 — 검색·언어 기준)
- (선택) `brand-gem`의 브랜드 보이스 / `competitor-quant`의 기회 키워드

## 실행 절차 (Opal 노드 → 스킬 단계)

### STEP 1 — 키워드·주제 리서치 (Semrush MCP 강화)
- Semrush로 타깃 국가의 메인 키워드 **검색량·KD·연관 키워드·질문형(Questions)** 조사.
- 구글 SERP의 'People Also Ask' 상위 3개 또는 AI Overview 인용 콘텐츠 주제도 함께 확보.
- → 최종 주제 리스트 3개 도출.

### STEP 2 — 아웃라인 생성
- 주제 리스트로 `<h1><h2><h3>` 구조의 논리적 목차 작성. 각 섹션 핵심 내용 요약 포함.

### STEP 3 — 본문 작성 (SEO + E-E-A-T)
- 분량 1,500~2,000단어, **타깃 국가 언어**로 작성.
- HTML 바디(`<h1>~<h3>`)만, `<head>` 생략. 중학생도 읽을 가독성.
- 메인 키워드 밀도 **전체 5% 이내** 자연스럽게 배치.
- **E-E-A-T 강화**: 실제 사례·최신 통계 근거 포함, 전문가적 견해.
- **GEO 최적화**: 각 H 태그 뒤 첫 40~60단어에서 제목에 즉답, 리스트·FAQ·정의 리드 문장 활용.

### STEP 4 — 번역
- 원문 블로그를 한국어로 번역(HTML 태그 구조 유지, 자연스러운 문맥). 원문/번역 구분 출력.

### STEP 5 — SEO 메타데이터
- 클릭 유도 **타이틀 태그** + 150자 내외 **메타 디스크립션**. 한글 + 타깃 국가 언어 두 버전.

## 산출물
1. 최종 주제 리스트(검색량 근거)
2. HTML 블로그 본문 (타깃 언어) + 한국어 번역본
3. 타이틀 태그 + 메타 디스크립션 (2개 언어)

## 🔗 핸드오프
- ← `competitor-quant`(기회 키워드), `brand-gem`(브랜드 보이스/톤)
- Opal 워크플로우(노드 편집형)와 **상호 보완**: 수준·취향에 따라 선택.

## 작성 원칙
- 장황한 키워드 채우기는 금물(SEO·GEO 모두 싫어함). 고유 관점·근거를 더한다.
- 전통 SEO(백링크·내부링크·콘텐츠 최적화)도 병행 — AI 플랫폼도 결국 검색엔진에 의존.
- AI는 초안을 생성할 뿐, 사실 검증과 고유 인사이트는 사람이 더한다.
