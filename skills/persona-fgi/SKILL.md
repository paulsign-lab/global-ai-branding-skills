# persona-fgi — 가상 FGI 페르소나·질문 설계

레딧이 "고객이 **먼저** 한 말(관찰)"이라면, 가상 FGI는 "그 고객에게 **되묻는 것**(대화)"이다.
이 스킬은 `reddit-voc`가 실데이터로 만든 페르소나 원형과 질문 어젠다를 받아, 검증
가능한 FGI 질문 세트로 완성한다. 핵심은 **상상으로 페르소나를 만들지 않고, 레딧
데이터에 기반해 만들어 합성 페르소나의 동조편향·일반화를 보정**하는 것.

## 언제 사용하나
- `reddit-voc` 다음 단계로, 가설을 페르소나에게 직접 물어 검증할 때
- AskRally 등 가상 FGI 도구에 넣을 페르소나·질문지를 준비할 때
- 미션/포지셔닝/스토리 설계 전, "FGI 결과" 입력값을 만들 때

## 입력값 (앞 스킬 산출물 우선)
- **`reddit-voc`의 브릿지 산출물** — 페르소나 원형 3~5개 + FGI 질문 어젠다 (필수 입력)
- **`competitor-quant`의 페인포인트 키워드 TOP5** (2축 검증용)
- 산업/카테고리, 우리 브랜드명, 진출 시장, 경쟁 브랜드 2~3개, 원산지, 핵심 차별점, 현재 단계

## 실행 절차

### STEP 1 — 페르소나 확정 (레딧 그라운딩)
`reddit-voc`의 페르소나 원형을 FGI용 5명으로 확정한다. 각 페르소나에
(a) 인구통계·피부고민/관심사, (b) **레딧 실제 표현 인용 2~3개**, (c) 구매 맥락을 붙인다.
→ "상상이 아니라 데이터에서 온 페르소나"임을 명시.

### STEP 2 — 4축 20문항 질문 설계
아래 4축 × 각 5문항. 각 문항은 ▸질문 / ▸의도 / ▸후속질문 형식.
- **1축 현재 행동(As-Is)**: 정보탐색·구매 채널, 현재 사용 제품과 이유, 구매빈도·지출, 의사결정 기준 우선순위
- **2축 페인포인트 검증**: ← `reddit-voc`/`competitor-quant`의 TOP 페인포인트를 직접 검증
  (가장 큰 불만, 안 쓰게 된 경험, 이탈 계기, 불만 발생 맥락+그 후 행동)
- **3축 메시지 테스트**: ← `reddit-voc`의 마케팅 카피 후보 A/B 비교, 신뢰가는 약속 vs 의심받는 약속, 원산지 인식
- **4축 진입장벽·전환 트리거**: 첫 구매 최소조건, 카테고리 특유 장벽, 전환 촉진 수단, 충성 브랜드 이탈 계기

### STEP 3 — 추가 산출
- **카테고리 특수 질문 3개** (이 산업에서만 의미 있는 질문)
- **페르소나 교차 분석 가이드** (답변 일치=공통 인사이트 / 갈림=세그먼트 전략)
- **FGI 결과 → 브랜딩 연결 매트릭스** (각 답변이 어떤 브랜딩 요소에 반영되는지)

## 산출물
1. FGI 페르소나 5명 정의서 (레딧 근거 포함) — AskRally 입력용 CSV/표
2. 4축 20문항 질문지 (의도·후속질문 포함)
3. 카테고리 특수 질문 + 교차 분석 가이드 + 연결 매트릭스

## 🔗 핸드오프
- ← **`reddit-voc`**: 페르소나 원형·질문 어젠다를 입력으로 받음 (연결의 핵심).
- → **`brand-mission`/`brand-positioning`/`brand-story`**: FGI 결과(5명 공통 발견,
  가장 감정이 강했던 발언 3개, 구매 기준 우선순위, 원산지 인식)가 각 설계 프롬프트의
  `■ 페르소나 FGI 결과` 입력값으로 그대로 들어간다.

## 작성 원칙
- 가상 FGI는 **방향성 검증·가설 수립용**이다. 실제 인간의 비합리적 감정·문화적 뉘앙스를
  완벽히 재현하지 못하며(동조편향 주의), 최종 확인은 실제 소규모 FGI로 넘긴다.
- 페르소나의 모든 속성은 레딧/정량 데이터에 근거를 둔다(상상 금지).
- AI는 질문을 설계하고 시뮬레이션할 뿐, 최종 해석은 사람의 몫.
