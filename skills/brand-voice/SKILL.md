# brand-voice — 브랜드 일관성 콘텐츠 엔진

콘텐츠를 만들 때마다 톤이 흔들리는 문제를 없앤다. 앞에서 만든 브랜드 자산(철학·톤앤매너·
페르소나·메시지 체계)을 **하나의 보이스 규칙**으로 고정해, AI가 단순 글쓰기 도구가 아니라
'일관된 브랜드 페르소나를 가진 전속 직원'처럼 일하게 한다.

> 기존 강의에서는 이 역할을 **Gemini Gems**로 세팅했다. 이 스킬은 같은 목적을
> Claude Code 안에서 네이티브하게 처리한다(별도 Gem 없이도 일관성 유지). Gemini를
> 쓰고 싶을 때를 위해 Gem용 시스템 프롬프트도 함께 출력할 수 있다.

## 언제 사용하나
- 브랜드 전략(미션~스토리) 확정 후, 콘텐츠를 일관된 톤으로 대량 생산할 때
- 인스타·블로그·CS 등 비SEO 콘텐츠를 온브랜드로 뽑을 때
  (SEO 블로그 콘텐츠는 `seo-content`가 이 보이스 규칙을 가져다 쓴다)

## 입력값 (★ 앞 스킬 산출물 연결)
- **브랜드 매니페스토 · 태그라인 · 메시지 체계** ← `brand-story`
- **톤앤매너 키워드 + 반드시 사용할 표현 / 절대 금지 표현**
- **페르소나 정의서** ← `persona-fgi` / `reddit-voc`
- **모범 콘텐츠 3~5개** (기존에 가장 잘 쓰인 글) — Few-shot 학습용(가장 중요)

## 실행 절차

### STEP 1 — 브랜드 보이스 규칙 고정 (3블록)
1. **[역할과 목적]** — 어떤 콘텐츠든 아래 규칙을 절대 준수해 일관된 브랜드 경험 제공.
2. **[브랜드 철학과 규칙]** — 페르소나, 핵심 철학/세계관, 톤앤매너, **사용할 표현**,
   **절대 금지 표현**(예: "최저가", "떨이", 비속어).
3. **[Few-shot 예시]** — 모범 콘텐츠 3~5개를 (요청→응답)으로 제공. AI가 '어떻게
   써야 하는지'를 구체적으로 학습(품질을 좌우하는 핵심).

### STEP 2 — 콘텐츠 생성
고정된 보이스 규칙으로 요청별 콘텐츠를 생성하고, 톤·금지표현 위반 여부를 자체 점검한다.
- 인스타그램 피드 / 블로그 에세이 / CS 스크립트 / 제품 설명 등

### (선택) STEP 3 — Gemini Gems 시스템 프롬프트 출력
Gemini를 쓰고 싶을 때를 위해, 위 3블록을 Gems용 '시스템 명령어' 형식으로도 출력한다.

## 산출물
1. **브랜드 보이스 규칙서** (3블록) — 재사용 가능한 콘텐츠 생성 기준
2. 요청별 샘플 콘텐츠 (인스타·블로그·CS 등) + 적합성 체크
3. (선택) Gemini Gems용 시스템 프롬프트

## 🔗 핸드오프
- ← `brand-story`(메시지 체계·매니페스토), `persona-fgi`(페르소나), 톤앤매너 파일
- → **`seo-content`**: 이 보이스 규칙이 SEO 콘텐츠 생성 시 톤 가이드로 적용된다.

## 작성 원칙
- Few-shot 예시가 품질을 좌우한다 — 모범 콘텐츠를 반드시 포함한다.
- 금지 표현을 명시한다.
- 전략·철학은 사람이 정한 것을 따른다 — AI는 그 규칙을 일관되게 실행할 뿐.
