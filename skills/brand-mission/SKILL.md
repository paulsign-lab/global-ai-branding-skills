# brand-mission — 브랜드 미션 & 비전 설계

데이터를 브랜드로 바꾸는 전환의 첫 단추. 미션은 **"우리가 매일 하는 일"**(현재형),
비전은 **"그 일을 계속하면 만들어질 미래"**(미래형)다. 이 스킬은 앞 파트에서 MCP로
모은 정량·정성 데이터를 재료로, 검증 가능한 미션·비전을 도출한다.

## 언제 사용하나
- 경쟁사 페인포인트·페르소나 데이터가 모인 뒤 브랜드의 '왜'를 정의할 때
- 캠페인마다 메시지가 흔들리는 문제를 막을 일관된 방향을 잡을 때

## 입력값 (★ 앞 스킬 산출물이 그대로 들어간다 — 개연성의 핵심)
- 브랜드명 / 카테고리 / 진출 시장 / 원산지 / 핵심 차별점
- **경쟁사 페인포인트 TOP 5 + 검색량** ← `competitor-quant` 산출물
- **페르소나 FGI 핵심 인사이트** (5명 공통 발견 + 가장 감정이 강했던 발언 3개)
  ← `persona-fgi` / `reddit-voc` 산출물
- **톤앤매너 키워드** (예: Clean, Scientific, Warm)

> 이 입력값들은 임의로 짓는 게 아니라, 앞에서 Semrush·Reddit·AskRally MCP로
> *실제 수집·검증한 데이터*다. 그래서 미션이 직감이 아니라 근거 위에 선다.

## 실행 절차

### Brand Mission — 후보 3개
공식: **"우리는 [누구]를 위해 [어떤 행동]을 통해 [어떤 변화]를 만든다"**
각 후보에 포함:
- 미션 문장 (공식대로)
- [누구]: 페르소나 데이터에서 도출
- [어떤 행동]: 실제 비즈니스 활동과 직접 연결
- [어떤 변화]: 경쟁사 페인포인트 중 어떤 문제를 해결하는지
- **경쟁사 이름 바꾸기 테스트**: 미션의 브랜드명을 경쟁사로 바꿔도 성립하면
  → 차별화 실패 → 반드시 수정
- → 추천안 1개 선택 + 이유

### Brand Vision — 후보 3개
공식: **"우리는 [어떤 세상/미래]를 만들고 싶다"**
각 후보에:
- 비전 문장
- 위 미션을 10년 실행하면 이 비전에 도달하는지 연결 설명
- 비전 달성을 알 수 있는 신호 1가지
- → 추천안 1개 선택 + 이유

## 산출물
- 미션 후보 3개 + 추천안 1개(+근거)
- 비전 후보 3개 + 추천안 1개(+근거)
- 경쟁사 이름 바꾸기 테스트 통과 여부 명시

## 🔗 핸드오프
- ← `competitor-quant`(페인포인트 검색량), `persona-fgi`(FGI 발언·인사이트)
- → **`brand-positioning`**: 확정된 미션·비전이 포지셔닝 설계의 입력값.
- → **`brand-story`**: 확정 미션·비전이 오리진 스토리의 [결심]·[약속]으로 연결.

## 작성 원칙
- 미션이 "싸게 팔겠다"류의 가격 전략이 되지 않게 한다(감정적 상위 미션 필요).
- 경쟁사 이름 바꾸기 테스트를 반드시 통과시킨다.
- 최종 선택은 사람이 한다 — AI는 검증 가능한 후보를 제시할 뿐.
